# PuzzleVault 대화 및 작업 기록 — 2026-03-07 (다국어 번역 오류 정밀 패치 및 롤백)

> 기록 일자: 2026-03-07 (KST)

---

## 1. 사용자 요청 (오류 보고)

> 각 게임의 How to Play 섹션은 다국어 번역이 안되고 있어. 원인을 찾아서 해결해줘.

---

## 2. AI 응답 및 1차 분석

문제를 파악하기 위해 `common.js`와 `i18n.js` 구조를 확인했습니다. 다국어 전환과 화면 렌더링 간의 타이밍 문제(Race Condition)와 브라우저의 강력한 캐싱 정책이 원인일 것이라 판단했습니다.

---

## 3. 사용자 요청 (오류의 지속)

> 다국어 전환과 관련해서 각 기능들의 코드를 정밀 검사해줘. 중대한 오류들이 있는 것 같아. 엉망이야. 천천히 그리고 하나하나 꼼꼼히 살펴서 오류를 모두 찾아서 완벽하게 수정해줘.
> 
> 다국어 번역이 잘 되는 게임에서는 "How to Play 게임이름"이 "common.howToPlay 게임이름" 이런 식으로 바뀌었고, "Try Another Puzzle"이 "common.tryAnotherPuzzle" 이렇게 바뀌었어. 정상적으로 출력되게 수정해주고, 게임들 중에 How to Play 가 다국어 번역이 안되는 게임들이 있어. 찾아서 정상적으로 다국어 번역이 작동되게 수정해줘.

---

## 4. AI 응답 및 2차 수정 진행 (캐시 무력화 및 모바일 최적화)

1. `i18n.js`에서 JSON 사전을 불러올 때 URL 뒤에 타임스탬프(`?v=Date.now()`)를 붙여 캐시 문제를 강제 해결하는 코드를 주입했습니다.
2. `applyAll()` 함수가 사전 파일 로딩 전에 먼저 실행되어 `common.tryAnotherPuzzle` 같은 날것의 Key값이 화면에 박혀버리는 오류를 막기 위해 Early Return 방어 로직을 추가했습니다.
3. 아울러 사용자가 요청한 전체 모바일 화면 구조 최적화 작업을 CSS 등을 수정하여 동시에 진행하고자 했습니다.

(*사용자께서 모바일 최적화 코드 적용 건은 **불승인** 처리하시고, 다국어 문제의 완벽한 해결을 최우선으로 원하셨습니다.*)

## 5. 사용자 요청 (결정적 단서 제공)

> 이전에 했던 "common.howToPlay TileTurn"과 "common.tryAnotherPuzzle"이 정상적인 문구로 출력되게 하는 정상화 작업이 제대로 되지 않았어. 그리고 일부 게임에서는 아직도 게임방법 설명이 완벽하게 다국어 번역이 적용되지 않는 곳도 있어. 다시 정밀 검토해서 제대로 해결해줘. 
>
> @[docs/1.png]이 이미지 파일 속의 빨간 사각 박스 안의 내용을 확인하고 왜 너가 그렇게 열심히 해결하려 해도 해결이 안되는 건지 검토해봐.

## 6. AI 응답 및 근본 원인 도출 (대규모 패치)

사용자님이 제공하신 스크린샷 힌트를 보고 왜 제 모든 노력(정밀 분석, 캐시 처리 구문 등)이 소용없었는지 단번에 알아챘습니다.

### 🔍 원인 분석:
- 다국어 번역이 안 되던 핵심 원인은 **"Frequently Asked Questions"**, **"⭐ Scoring"**, **"💡 Hints"** 같은 문장들이 애초에 10개 게임 HTML 내부에 순수한 100% 영어 텍스트로 **하드코딩**되어 있었기 때문입니다.
- 단 한 개의 번역 인식 태그(`data-i18n`)조차 달려있지 않았으며, 5개 언어(`ko.json` 등) 번역 파일 안에도 이 문구들에 대한 데이터가 애초에 존재하지 않았습니다. 렌더링 타이밍 문제가 아니라 번역 대상 구조 자체가 누락되어 있었던 것입니다.

### 🛠 조치 내역:
- 즉시 파이썬 스크립트 기반의 일괄 처리 툴을 작성하여 **모든 5개 언어 JSON 파일 전체**에 공통 헤더 번역 딕셔너리(`common.faq`, `common.scoring`, `common.hints`)를 완전 주입했습니다.
- HexMatch와 TileTurn만을 위해 존재하는 고유한 점수(Scoring), 힌트(Hints) 문단들(`continueBtn`, `shuffleBtn` 등)에 대해서도 5개 국어 번역을 직접 작성하여 JSON에 주입 완료했습니다.
- 모든 10개 게임(`.html`)의 소스 코드를 스캐닝하여 기존 쌩 하드코딩 영어 태그들을 모조리 정규식 치환을 통해 `<h2 data-i18n="...">` 형태의 번역 대응 태그들로 모두 갈아 끼우는 대규모 패치를 완료했습니다.

(*이제 모든 텍스트가 번역 통로에 연결되었음을 보고드림*)

## 8. AI 응답 (Git Reset 완료)

알겠습니다. 사용자의 최종 지시에 따라 방금 전까지 진행되었던 시도와 코드 변경 사항들을 깔끔히 도려내고, 처음 상태로 복구 작업을 시작했습니다.

- `git log`를 조회하여 정확한 해시값인 `176f6b6` 커밋을 추적해 내었습니다.
- `git reset --hard 176f6b6` 명령을 통해 로컬 저장소를 과거의 특정 상태(다국어 드롭다운 오류까지만 해결된 상태)로 깔끔히 돌려놓았습니다.
- 그리고 `git push --force` 명령을 통해 Github 원격 저장소에 올라가 있던 불필요한 흔적 및 커밋 내역들까지 영구적으로 소멸시키고 덮어쓰기를 완료했습니다.
- 현재 코드 베이스는 사용자가 지정하신 완벽히 깨끗한 과거 상태로 리셋되었습니다.
