사용자님이 제공하신 스크린샷 힌트를 보고 왜 제 모든 노력(정밀 분석, 캐시 처리 구문 등)이 소용없었는지 단번에 알아챘습니다.

- 다국어 번역이 안 되던 핵심 원인은 **"Frequently Asked Questions"**, **"⭐ Scoring"**, **"💡 Hints"** 같은 문장들이 애초에 10개 게임 HTML 내부에 순수한 100% 영어 텍스트로 **하드코딩**되어 있었기 때문입니다.

### 🛠 조치 내역:
- 즉시 파이썬 스크립트 기반의 일괄 처리 툴을 작성하여 **모든 5개 언어 JSON 파일 전체**에 공통 헤더 번역 딕셔너리(`common.faq`, `common.scoring`, `common.hints`)를 완전 주입했습니다.
- HexMatch와 TileTurn만을 위해 존재하는 고유한 점수(Scoring), 힌트(Hints) 문단들(`continueBtn`, `shuffleBtn` 등)에 대해서도 5개 국어 번역을 직접 작성하여 JSON에 주입 완료했습니다.
- 모든 10개 게임(`.html`)의 소스 코드를 스캐닝하여 기존 쌩 하드코딩 영어 태그들을 모조리 정규식 치환을 통해 `<h2 data-i18n="...">` 형태의 번역 대응 태그들로 모두 갈아 끼우는 대규모 패치를 완료했습니다.

## 12. AI 응답 (심층 디버깅 및 치명적 버그 적발/완전 패치, 브라우저 캐시 v8 업데이트)

사용자께서 연달아 제보해주신 여러 잔존 버그들을 파헤치기 위해, 표면적인 코드가 아닌 심연의 로직들을 철저히 파악하여 다음과 같은 핵심 문제들을 해결했습니다.

### 🐛 1. 다국어 키값 노출(Tagline 등) 및 렌더링 무반응 (캐시 문제 파악)
- **원인:** 번역 엔진 로직을 아무리 완벽하게 치환 코드로 교체해 두어도 사용자의 브라우저 내부 백그라운드 **서비스 워커(sw.js)**에서 낡은 오프라인용 캐시 자바스크립트(`common.js` 등)를 강제로 로드하고 있어 패치가 화면에 나타나지 않음.
- **조치:** `sw.js`의 캐시 버전 식별자를 강제로 대폭 상향(`v4` -> `v5, v6, v7, v8`)하여 모든 클라이언트가 구버전 파일들을 강제로 파기하고 최신 파일을 수신하도록 무효화 정책 시행. 동시에 `index.html` 하단 추천 게임 영역 및 게임 내 하단 크로스 프로모션 UI의 제목 태그(`<span data-i18n="common.tryAnotherPuzzle">`)와 플레이 버튼 번역 바인딩 구조 결함 재수정 성공.

### ⏱️ 2. 공통 타이머 속도 2배/3배 비정상 가속 (Falsy 버그)
- **원인 (NumVault, GridSmash, MergeChain 등 100% 공통):** 자바스크립트 엔진이 랜덤으로 반환하는 `Interval ID 식별자 값`이 0으로 배정되는 경우를 생각하지 못했음. 기존 코드(`if (G.speedInterval) clearInterval...`)가 식별자 0을 `false` 단어로 해석하여 타이머를 해제시키지 않고 무시해버리는 **Falsy 비교 연산 오류(블랙홀 버그)**가 존재했음.
- **조치:** 모든 구간의 타이머 해제 조건을 `!== null` 형식으로 정교하게 우회 비교하여, 재시작을 아무리 여러번 눌러도 오직 단일 스레드의 타이머 1개만 돌도록 완벽 봉쇄함.

### ⚡ 3. QuickCalc 게임 다중 클릭 스팸 및 연산 오류 대방출
**① 심각한 콤보/문제 수 폭주 버그 (Click Spam):**
- 정답을 맞춘 뒤 다음 문제 애니메이션 대기 시간(약 120ms) 동안 손이 빠른 사용자가 정답 버튼을 무작위로 연타하면, 1문제 당 10배~20배가 넘는 상상 초월의 점수, 콤보, 문제 번호 스킵이 발생하는 치명적 방어 로직 누수가 존재했음.
- **해결:** `state.waitingForNext` 안전 스위치 변수를 생성하여 정답을 누르는 순간 켜지도록 코딩. 새 카드가 뜨기 전까지 그 어떤 터치 입력도 절대로 수용되지 않게 방어막 적용.

**② 탭(모드) 전환 완전 무시 오류:**
- 게임 도중 실수로 다른 탭 패널(Time Attack 등)로 눌러 이탈하는 것을 막기 위한 강제 차단 코드(`if (state.isPlaying) return;`) 탓에 진행 중에는 탭 변경이 영구 금지되어 있었음.
- **해결:** 강압적인 차단 룰을 지우고, 대신 "진행 중인 게임을 종료하고 다시 시작할 것인가요?"(confirm 모달)라는 매너 있는 안내 창을 띄워 게임을 안전하게 리셋시키고 새 탭으로 유유히 이동할 수 있도록 개선 완료.

**③ 고난이도(Phase 4) 수학 사칙연산 우선순위 파괴 오류:**
- `18 + 3 × 3` 과 같은 높은 레벨의 수식이 출제될 경우 엔진은 `(18 + 3) × 3` 이라는 앞에서부터 단순 순차 누적 계산 방식으로 엉터리 오답 66을 정답이라 요구하고 있었음.
- **해결:** `generateProblem` 문제 도출 코드에 사칙연산의 제1 규칙인 곱셈 우선 판단 IF 문을 삽입, `b * c`를 선 계산한 뒷 값을 a에 더하도록 논리 수식 공식 완벽 재수립. (참고로 나눗셈은 1-step용으로 소수점 없이 강제 도출되게끔 사전 통제되어 안전함)
