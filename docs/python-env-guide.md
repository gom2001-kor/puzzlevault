# 파이썬 환경 및 VS Code 인터프리터 설정 가이드

본 문서는 VS Code 환경에서 파이썬(Python) 기반의 풀스택 프로젝트 또는 파이썬 스크립트를 포함하는 프로젝트 진행 시 나타나는 `Select Python Interpreter` 문제 해결 방안과 실무에서의 권장 환경 설정 방식을 안내합니다.

## 1. 'Select Python Interpreter' 메시지의 원인 및 해결

VS Code가 파이썬 코드를 실행하거나 분석하기 위해 시스템 내의 어느 파이썬 환경(버전 및 패키지)을 사용해야 할지 알지 못할 때 발생하는 자연스러운 현상입니다.

### 📌 해결 방법 (프로젝트 환경에 파이썬 인터프리터 연동)
1. **VS Code 명령어 팔레트 실행:** `Ctrl + Shift + P`
2. **명령어 검색 및 선택:** `Python: Select Interpreter` 입력 후 클릭
3. **환경 선택:** 드롭다운 목록에서 시스템에 설치된 Python (예: `Python 3.1x.x`) 또는 가상 환경(`.venv`) 선택
4. **결과:** 이 작업을 완료하면 프로젝트 내부 `.vscode/settings.json` 파일에 `"python.defaultInterpreterPath"` 값이 자동으로 저장되고, 다시는 알림이 표시되지 않습니다.

---

## 2. 순수 프론트엔드 프로젝트(HTML/CSS/JS)에서의 파이썬 설정

현재 진행 중인 **PuzzleVault 프로젝트**와 같이 100% 클라이언트 사이드 프론트엔드로만 구성된 프로젝트의 경우:

* **부작용 없음:** 이 설정은 VS Code라는 '에디터'에게 파이썬 파일의 위치를 알려주는 설정일 뿐, 실제 웹 서비스 코드나 배포 환경에는 아무런 영향을 주지 않습니다.
* **사용 목적:** 로컬 서버 구동 (예: 파이썬 모듈을 이용한 간단한 서버 띄우기) 등 부가적인 터미널 스크립팅을 할 때 에디터의 불필요한 알림 창(Select Python Interpreter)을 끄기 위한 목적이 큽니다.

---

## 3. 파이썬 풀스택 프로젝트에서의 실무 권장 방식 (가상 환경)

Django, FastAPI, Flask 등을 활용한本格적인 파이썬 백엔드 또는 풀스택 프로젝트를 새로 시작할 경우, 전역 파이썬 대신 **가상 환경(Virtual Environment)**을 사용하는 것이 필수적입니다.

### ✅ 풀스택 프로젝트 시 인터프리터 설정이 주는 이점
1. **IntelliSense (자동 완성 및 문법 검사):** 설치된 외부 프레임워크나 라이브러리의 코드를 에디터가 인식하여 자동 완성을 정확히 지원합니다.
2. **원활한 디버깅:** 브레이크포인트(Breakpoint)를 통한 코드 레벨 변수 추적 등 VS Code의 내장 디버그 기능을 에러 없이 사용할 수 있습니다.
3. **코드 포매팅/린팅:** Black, Ruff 등의 파이썬 전용 코드 정리 도구를 원활하게 연동할 수 있습니다.

### 🚀 새 프로젝트 진행 시 설정 순서
1. **가상 환경 생성:** 
   프로젝트 루트 터미널에서 다음 명령어 중 하나를 실행하여 `.venv` 폴더를 생성합니다.
   * 기본 방식: `python -m venv .venv`
   * uv 사용 시: `uv venv`
2. **VS Code에 가상 환경 연동:**
   * VS Code 화면 우측 하단의 가상 환경 감지 알림에서 "예(Yes)" 선택.
   * 또는, 명령어 팔레트(`Ctrl + Shift + P`) -> `Python: Select Interpreter` -> 생성된 가상 환경 경로 (예: `.\.venv\Scripts\python.exe`) 선택.
3. **설정 완료 확인:**
   자동으로 업데이트된 `.vscode/settings.json` 파일을 통해, 에디터는 프로젝트 전용 격리된 파이썬 환경을 안정적으로 바라보게 됩니다.
