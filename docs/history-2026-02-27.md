## 2. 블로그 시스템 구축 및 오류 수정 내역 (현재는 롤백되어 존재하지 않음)

**⚠️ 주의: 이 아래의 작업들은 사용자의 추가 요청에 의해 모두 취소(Rollback) 및 삭제되었으며, 리포지토리(GitHub 포함)에서 완전히 지워졌습니다. 현재 코드베이스에는 반영되어 있지 않습니다.**

1. **PuzzleVault 전용 블로그 시스템 셋업**
   - **작업명:** "PuzzleVault 블로그(Blog) 시스템 구축"
   - **내용:** `js/blog-data.js`를 만들어 블로그 정보(`BLOG_POSTS` 메타데이터 및 필터링 헬퍼 함수)를 관리하고, `blog/index.html` (카테고리 탭, 그리드 카드, Load More 버튼)과 공통 템플릿인 `blog/posts/_template.html`을 개발. 추가로 이에 맞는 3개의 예시 글을 생성하여 배포하고 URL을 `sitemap.xml`에 추가했습니다. 이와 동시에 `common.js`에 블로그용 네비게이션 항목과 메인 페이지 홈카드 용 `renderBlogCards()` 함수를 추가했습니다.
   - **초기 보고서 작성:** 위 모든 내용을 `docs/work-report-2026-02-27.md` 문서로 정리해 보고했습니다.

2. **Service Worker (`sw.js`) 캐싱 버그 및 메인 화면 Footer 경로 문제 해결**
   - **이슈:** `About` 페이지 접속 시 "사이트에 연결할 수 없음" 메시지가 뜨고 오류가 남.
   - **문제 원인 1 (SW 캐싱):** `sw.js`에서 캐시 누락(혹은 인터넷 연결 차질) 시 적절한 네트워크 `fetch` Fallback과 예외처리가 부족하여 브라우저에 `undefined`를 던져 페이지 로딩이 마비되는 문제였음.
   - **문제 원인 2 (절대 경로 오류):** `common.js`의 Footer 생성 함수 안에 있는 링크들(`href="/about.html"`)이 절대경로라서, 로컬 환경(Live Server 하위경로나 `file:///` 프로토콜)에서 웹서버 최상단 경로를 찾지 못해 터지는 버그였음.
   - **조치:** `sw.js` 캐시 `Promise.allSettled()` 리팩토링 및 캐시v2 버저닝. `common.js` 내부에 현재 경로 깊이를 계산해 상태 경로로 바꿔주는 `resolvePath()` 함수 제작 및 적용, 더불어 localhost 테스트 시 혼선을 막기 위한 Service Worker 자동 Unregister 로직도 추가했습니다.

## 3. 작업 전면 롤백 (Rollback) 및 히스토리 정리 (최종 상태)

1. **작업 초기화 요청과 실행**
   - **요청:** `"모든 지원 페이지(about, privacy, terms, contact)와 사이트맵(sitemap.xml), robots.txt 업데이트, 그리고 Cloudflare Pages를 위한 URL 리다이렉트 설정(_redirects) 작성을 깔끔하게 완료했습니다."` 라는 커밋 메시지의 상태로 코드를 되돌리고, 로컬과 깃허브에서 이후 내용을 깨끗이 지워라.
   - **조치:**
     - `git log` 명령어를 통해 해당 커밋(`1a982f0`) 확인.
     - `git reset --hard 1a982f0` 을 통해 깃 히스토리 영구 역산 및 롤백.
     - `git clean -fd`와 수동 파일 제어(`git rm -r blog js/blog-data.js`)를 통해 추적되지 않거나(`UNTRACKED`) 잔재하던 블로그 관련 신규 파일들을 강제 삭제 및 청소.
     - 이 과정에서 `index.html`, `js/common.js`, `css/global.css`, `sw.js`, `sitemap.xml`, `_redirects` 파일 안에서 블로그 기능만 따로 도려내며 롤백을 마무리함.
     - **`git push origin main --force`** 를 실행하여, 엉켜있던 GitHub의 원격 브랜치 히스토리를 강제 덮어씌워 깔끔한(`1a982f0` 기준) 커밋으로 통합 완료.
