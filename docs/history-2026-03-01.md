### 로컬 개발 시 주의사항
- SW가 캐싱으로 인해 변경 사항이 반영되지 않을 수 있음
- `navigator.serviceWorker.register('/sw.js')`는 절대 경로이므로 로컬 서버(Live Server 등) 사용 권장
- 변경 후 브라우저 DevTools → Application → Service Workers → Unregister 후 새로고침

## ColorFlow 색 타일 클릭 시 Undo 버그 수정

### 요청 내용

- ColorFlow 게임에서 색 타일(마커)을 클릭 또는 터치하면 기존 경로가 사라지는 undo 효과 발생

### 원인 분석 (2가지)

#### 1. 마커 클릭 시 즉시 경로 삭제
- `handlePointerDown()` (345행)에서 색 마커 터치 즉시 `clearPath(col)` 호출
- 단순 탭이나 실수 터치에도 공들여 그린 경로가 통째로 삭제

#### 2. 이벤트 리스너 중복 등록
- `buildBoardDOM()`이 레벨 전환마다 호출되는데, `grid.addEventListener` 4개가 매번 누적 등록
- 한 번 클릭해도 여러 번 처리됨

### 수정한 파일

#### `games/colorflow-logic.js` — 4개 함수 수정

**1. `buildBoardDOM()` — 이벤트 중복 등록 방지**
- `grid.addEventListener` 앞에 `grid.removeEventListener` 4줄 추가
- `pointerdown`, `pointermove`, `pointerup`, `pointercancel` 모두 처리

**2. `handlePointerDown()` — 마커 클릭 시 즉시 clearPath 제거**
- 수정 전: `saveSnapshot()` → `clearPath(col)` → `state.paths[col].push(idx)` 즉시 실행
- 수정 후: `state._pendingClear = true` 플래그만 설정, 실제 삭제는 드래그 시작 시까지 지연

**3. `handlePointerMove()` — 첫 드래그 시에만 deferred clear 처리**
- `state._pendingClear`가 true일 때만 `saveSnapshot()` + `clearPath(col)` + `state.paths[col].push(startIdx)` 실행
- 이후 정상적으로 경로 그리기 진행

**4. `handlePointerUp()` — 단순 탭 시 _pendingClear 리셋**
- `state._pendingClear = false` 추가
- 마커를 탭만 하고 드래그하지 않으면 기존 경로 보존

### 동작 변화

- **Before**: 마커 탭 → 즉시 경로 삭제 (= undo 효과)
- **After**: 마커 탭만 → 아무 변화 없음 / 마커 탭 + 드래그 → 기존 경로 삭제 후 새로 그리기 시작
